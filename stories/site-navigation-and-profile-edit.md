# Story: 全站導覽 Header 與個人資料編輯模式

## 說明
身為已登入使用者,我想要在任何頁面都能透過固定的導覽列切換到首頁/個人資訊/場地產生器/登入登出,並在個人資料頁以「檢視 → 編輯 → 儲存」的明確流程修改暱稱,以便不必每次回到首頁才能導覽,且不會誤觸編輯欄位。

## 驗收條件
- 在任何頁面下,當使用者查看畫面上方,則可看到固定的 Header,包含首頁、個人資訊、場地產生器的導覽連結,以及登入/登出狀態對應的操作(未登入顯示登入/註冊,已登入顯示個人資訊/登出)。
- 在個人資料頁載入時,當使用者查看畫面,則暱稱欄位為唯讀顯示,上方顯示「編輯」按鈕而非表單。
- 在個人資料頁的檢視狀態下,當使用者點擊「編輯」,則暱稱欄位變為可輸入,「編輯」按鈕變為「儲存」(可能同時提供「取消」)。
- 在個人資料頁的編輯狀態下,當使用者點擊「儲存」且暱稱驗證通過並成功送出,則畫面回到唯讀的檢視狀態,顯示更新後的暱稱與成功訊息。
- 在個人資料頁的編輯狀態下,當儲存失敗(驗證失敗或 API 錯誤),則畫面停留在編輯狀態並顯示錯誤訊息,不會誤回檢視狀態。

## 任務清單
- [x] [FRONTEND] 建立全站導覽 Header 元件:首頁/個人資訊/場地產生器連結 + 登入登出狀態(復用現有 AuthNav 登入狀態偵測邏輯),掛載到 RootLayout 讓所有頁面共用
- [x] [FRONTEND] 個人資料頁改為檢視/編輯模式切換:預設唯讀 + 「編輯」按鈕,點擊後進入可編輯狀態顯示「儲存」,成功儲存後返回唯讀狀態
