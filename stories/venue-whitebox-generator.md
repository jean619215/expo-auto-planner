# Story: 場地白模產生器 (階段一)

## 說明
身為使用者,我想要用專業平面圖風格的編輯器規劃場地,並自動產生對應的 3D 白模給我旋轉檢視,以便快速確認場地空間規劃是否合理。

> **改版備註 (2026-07-12)**: 原本的「網格格子圈選」編輯器(已完成的 task 1-3, commits 3afc5fc/e9cf950/0270f89)經使用者確認**整組換掉**,改為專業平面圖風格。以下為新方向的定調與任務清單。

架構定調 (與使用者討論後確認):
- 2D 編輯器改為**專業平面圖風格**,使用 **Konva.js (react-konva)** 實作 canvas 互動:
  - **地板範圍 = 多邊形**: 頂點可拖拉調整,可在邊上插入新頂點,形狀可不規則(凹凸)。
  - **牆壁 = 線段**: 拉一條線就是一面牆,固定厚度(0.2m),可拖曳兩端點調整長度與方向。
  - **柱子 = 小物件**: 矩形,拖曳擺放。
  - 物件(牆/柱)可**選取、拖曳移動、縮放**,縮放時**即時顯示尺寸**(公尺)。
  - 所有拖拉操作**吸附 0.5 公尺**刻度。
  - 畫面模擬平面圖外觀: 淺色底、格線參考、尺寸標註。
- 最大場地範圍 **50x50 公尺不變**,頂點/物件不可拖出界。
- 單位: 公尺,畫面上需標示真實尺寸(選取物件顯示尺寸、地板邊長標註)。
- 3D 白模(維持先前決策): Three.js + react-three-fiber;地板多邊形擠出成 0.1m 薄板(頂面 y=0),牆沿線段擠出、柱子為箱體,固定高度 3 公尺,不做門窗;配色比照 2D;一鍵「產生 3D 模型」,空場景時按鈕停用。
- 3D 檢視: orbit controls 旋轉/縮放。
- **本階段不做資料庫儲存** — 純前端單次編輯。
- 下一階段(本 story 不含): 3D 模型直接編輯、AI 輔助編輯。

## 驗收條件
- 使用者可以看到平面圖風格的編輯畫布(50x50m 範圍、格線參考、比例標示)。
- 使用者可以編輯地板多邊形: 拖拉頂點、在邊上新增頂點,形成不規則形狀;頂點吸附 0.5m。
- 使用者可以拉線段建立牆壁(固定厚度),拖曳端點調整;可以拖曳擺放矩形柱子。
- 使用者可以選取牆/柱物件後拖曳移動、縮放,縮放過程即時顯示尺寸(公尺);操作吸附 0.5m。
- 頂點與物件不可超出 50x50m 最大範圍。
- 使用者按下「產生 3D 模型」後,依當前平面圖產生 3D 白模(多邊形地板 + 沿線牆體 + 柱體,固定高度)。
- 3D 檢視畫面可以用滑鼠旋轉/縮放觀看整個白模。
- 編輯內容在離開頁面後不需保留(本階段無資料庫)。

## 任務清單
- [x] [FRONTEND] 建立 Konva 平面圖編輯器基礎:畫布、格線與比例標示、地板多邊形編輯(預設矩形起手,頂點拖拉、邊上插入頂點、0.5m 吸附、50x50 界限)
- [x] [FRONTEND] 物件系統:牆壁線段工具(固定厚度、端點拖拉)與柱子矩形工具(拖曳擺放),含選取/移動/刪除
- [x] [FRONTEND] 縮放與尺寸標註:物件縮放把手、縮放/選取時即時顯示公尺尺寸、地板多邊形邊長標註
- [x] [FRONTEND] 依平面圖資料建立 3D 白模(Three.js + react-three-fiber):地板多邊形擠出薄板、牆沿線段擠出、柱子箱體,固定高度 3m,「產生 3D 模型」按鈕
- [ ] [FRONTEND] 3D 檢視器整合:orbit controls 旋轉/縮放,2D 編輯與 3D 預覽切換流程
