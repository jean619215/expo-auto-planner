# Story: 場地白模產生器 (階段一)

## 說明
身為使用者,我想要用簡單的網格工具規劃場地平面,並自動產生對應的 3D 白模給我旋轉檢視,以便快速確認場地空間規劃是否合理。

架構定調 (與使用者討論後確認):
- 這是全新獨立功能,與「會員系統」story 無關,不共用其任務清單。
- 2D 編輯器採**網格格子圈選**模式(類似踩地雷),不是自由多邊形拖拉:
  - 使用者先定義最大外框(網格範圍)。
  - 用工具列在外框內的格子上作畫,依格子類型上色區分:地板 / 牆壁 / 柱子 / 空白(未圈選)。
  - 牆壁、柱子都各自是「一格」,與地板同一套網格系統,不是格子邊界上的線段 — 切換工具列(畫地板/畫牆壁/畫柱子/擦除)即可作畫,實作簡單。
  - 場地形狀可不規則 — 允許非連續、凹凸的地板圈選範圍。
  - 網格比例**固定**,每格代表 1 公尺,畫面上需標示真實尺寸(比例尺/座標軸標籤)。
- 3D 白模:依 2D 網格資料自動產生對應 3D 模型 —
  - 地板格子 → 延伸成地板平面。
  - 牆壁/柱子格子 → 依**固定高度**(不可調,不做門窗、不做造型變化)擠出成箱型量體。
  - 使用 Three.js + react-three-fiber (React 生態最成熟的 3D 方案) 產生/顯示模型。
  - 提供 orbit controls 讓使用者旋轉/縮放檢視 3D 白模。
- **本階段不做資料庫儲存** — 純前端單次編輯,產生 3D 預覽即可,使用者離開頁面編輯內容不保留。之後如果需要「回來繼續編輯/管理多個場地」再另外規劃儲存。
- 下一階段(本 story 不含,只是先讓大家知道方向): 3D 模型直接編輯、AI 輔助編輯。本階段完全不涉及。

## 驗收條件
- 使用者可以設定場地的最大外框尺寸(以公尺為單位,對應網格格數)。
- 使用者可以用「畫地板」工具在外框內任意圈選/取消圈選格子作為地板,允許不規則、不連續的形狀。
- 使用者可以用「畫牆壁」「畫柱子」「擦除」工具在格子上標記/清除牆壁、柱子。
- 畫面上不同類型的格子(地板/牆壁/柱子/空白)有清楚的顏色區分,且能看到網格對應的真實尺寸標示。
- 使用者按下「產生 3D 模型」後,依當前 2D 網格資料產生對應的 3D 白模(地板平面 + 固定高度的牆壁/柱子箱型量體)。
- 3D 檢視畫面可以用滑鼠旋轉/縮放觀看整個白模。
- 2D 編輯內容在使用者離開頁面後不需要保留(本階段無資料庫)。

## 任務清單
- [x] [FRONTEND] 建立 2D 網格編輯器基礎:渲染網格、設定外框尺寸(公尺數→格數換算)、格子點擊圈選/取消「地板」(先只有畫地板工具,無其他工具列)
- [ ] [FRONTEND] 擴充工具列:新增「畫牆壁」「畫柱子」「擦除」工具,可切換目前工具即時繪製對應格子類型,不同類型格子顏色區分清楚
- [ ] [FRONTEND] 網格真實尺寸標示(比例尺/座標軸公尺數標籤),確認不規則(非連續、凹凸)地板圈選範圍運作正常
- [ ] [FRONTEND] 依 2D 網格資料建立 3D 白模(Three.js + react-three-fiber):地板格子產生地板平面,牆壁/柱子格子依固定高度擠出成箱型量體
- [ ] [FRONTEND] 3D 檢視器整合:orbit controls 旋轉/縮放,2D 編輯畫面與 3D 預覽之間的切換/觸發流程
