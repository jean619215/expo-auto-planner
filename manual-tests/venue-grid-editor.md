# 手動驗證清單 — 場地白模產生器：2D 網格編輯器基礎（Task 1）

頁面：`/venue`

逐項於 `npm run dev` 起服務後於瀏覽器手動操作驗證。

1. **預設載入**：開啟 `/venue`，應立即看到 10×10 的網格，所有格子皆為「空」狀態（淺灰／白底）。
2. **有效尺寸調整**：將寬/高輸入為有效數值（例如 20 / 15）並按「套用尺寸」，網格應重新渲染為新尺寸，且先前畫的地板格全部清除。
3. **縮小尺寸**：先畫幾格地板，再把尺寸調整為比目前更小的數值，所有已選取的格子應全部被清除（而非只清除超出範圍的部分）。
4. **超過上限**：分別測試寬度 51、高度 51、以及寬×高組合超過 2,500 格（例如 50×51）的輸入，套用時應被拒絕，顯示上限提示訊息，且網格維持原本合法尺寸不變。
5. **無效輸入**：分別測試 0、負數、非數字、小數的寬/高輸入，套用時應被拒絕並顯示驗證訊息，網格維持不變。
6. **單擊切換**：單擊一個空格，應變成淺藍色地板格；再單擊同一格，應變回空格。
7. **拖曳繪製／擦除**：從空格開始按下拖曳過多個混合狀態的格子，所有經過的格子應變成地板；從地板格開始拖曳，所有經過的格子應變成空格。
8. **快速連續點擊**：對同一格快速連續點擊，應維持乾淨的單次切換（不應出現無反應或連續切兩次抵銷的情形）。
9. **放開／離開結束筆畫**：放開指標或指標離開網格範圍應立即結束繪製筆畫；未重新按下就重新進入網格範圍，不應繼續繪製。
10. **不連續形狀**：以多次分開的單擊或拖曳畫出不相連的地板區塊，形狀應如實保留，不會有自動連接、填滿或驗證行為。
11. **重新整理無持久化**：畫好地板後重新整理頁面，應恢復成預設 10×10、全空的狀態（不保留先前資料）。
